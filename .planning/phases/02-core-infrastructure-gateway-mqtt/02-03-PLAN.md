<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LAN client → /api/v1/auth/login | Untrusted clients submit username+password over HTTP. MVP targets local LAN (semi-trusted). |
| LAN client → /api/v1/* | Authenticated clients access protected API resources. Token must be presented on every request. |
| setup.sh → .env | Secrets written to disk with 644 permissions. |
| setup.sh → mosquitto_passwd | Hashed passwords generated on the deployment host. |
| Host filesystem → Docker container | .env and mosquitto_passwd are mounted into containers. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Spoofing | POST /api/v1/auth/login | mitigate | DRF TokenAuthentication validates username+password against Django auth backend. Brute force protection is deferred (rate limiting in hardening phase). |
| T-02-12 | Spoofing | API token in transit | accept | Token is sent in cleartext over HTTP (MVP on local LAN). HTTPS/TLS deferred to hardening phase. |
| T-02-13 | Information Disclosure | Token in URL/logs | mitigate | Token is sent via Authorization header, never in URL query params. DRF does not log tokens by default. |
| T-02-14 | Information Disclosure | .env file permissions | mitigate | setup.sh should set .env to 600 or 640 permissions. The file is in .gitignore. |
| T-02-15 | Elevation of Privilege | Django superuser auto-creation | mitigate | Superuser credentials are provided via .env file or setup.sh prompts. Only accessible on the deployment machine. entrypoint.sh creates superuser only if DJANGO_SUPERUSER_PASSWORD is set (opt-in). |
| T-02-16 | Tampering | setup.sh installation of mosquitto-clients | accept | Setup script runs apt-get install for mosquitto-clients. Package is from official Debian/Raspbian repos. This runs once on the user's machine. |
| T-02-SC | Tampering | mosquitto-clients pkg | mitigate | Installed from official OS package repos (apt). Treat as [ASSUMED] for MVP — no BLOCKING human checkpoint needed for standard OS packages. |
</threat_model>
