<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LAN → MQTT (1883) | Untrusted LAN clients connect to the broker. In v1, the LAN is considered semi-trusted (Raspberry Pi local network). |
| Container → Host | Mosquitto container can write to named volumes mosquitto_data and mosquitto_log. Named volumes isolate container writes. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Mosquitto listener 1883 | mitigate | allow_anonymous false + password_file auth via mosquitto_passwd. Unauthenticated clients are rejected at connection time. |
| T-02-02 | Information Disclosure | MQTT credentials in transit | accept | TLS is deferred to hardening phase. MVP runs on local LAN (semi-trusted network). Credentials are shared across all devices in v1 — a compromised credential affects all nodes. Communicated in /gsd-discuss-phase. |
| T-02-03 | Tampering | mosquitto_passwd file | mitigate | mosquitto_passwd is mounted read-only (:ro) into the container. Only the host (root) can modify it. The file is gitignored and generated by setup.sh. |
| T-02-04 | Tampering | Docker named volumes | accept | Named volumes are managed by Docker and isolated per-container. Single-user Raspberry Pi deployment — no multi-tenant risk. |
| T-02-SC | Tampering | eclipse-mosquitto:2 image | mitigate | Verified official Docker image. Use SHA-pinned digest in production (deferred to hardening). |
</threat_model>
