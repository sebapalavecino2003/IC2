    | Componente | Tecnología |
    |------------|------------|

    | Backend | Django 5, DRF, MySQL 8, Gunicorn, nginx |
    | Frontend | React 18, TypeScript, Vite, Material UI |
    | Broker MQTT | Mosquitto 2 |
    | Infraestructura | Docker Compose (Raspberry Pi / Linux) |

    ## Inicio Rápido

    ```bash
    # 1. Clonar el repositorio
    git clone <repo-url> && cd nodealert-iot

    # 2. Configurar variables de entorno (editar .env con tus credenciales)
    cp .env.example .env

    # 3. Ejecutar script de despliegue
    ./setup.sh

    # 4. Compilar firmware
    cd NodeAlert-Firmware && pio run && cd ..

    # 5. Iniciar servidores
    docker compose up -d
    ```

    **docs/API.md** (per D-17, API.md pattern):
    Document all REST endpoints organized by category + MQTT topics:

    **Autenticación:**
    - `POST /api/v1/auth/login/` — Login (username+password → token). Rate limit: 5/min por IP.

    **Dispositivos:**
    - `GET /api/v1/devices/` — Listar dispositivos (filtrable por is_active, location)
    - `POST /api/v1/devices/` — Crear dispositivo
    - `GET /api/v1/devices/{id}/` — Detalle de dispositivo
    - `PUT /api/v1/devices/{id}/` — Actualizar dispositivo
    - `PATCH /api/v1/devices/{id}/` — Actualización parcial
    - `DELETE /api/v1/devices/{id}/` — Eliminar dispositivo
    - `POST /api/v1/devices/{id}/command/` — Enviar comando MQTT

    **Lecturas (Read-Only):**
    - `GET /api/v1/readings/` — Listar lecturas (filtrable por sensor_type, device_id, timestamp__gte, timestamp__lte)

    **Eventos:**
    - `GET /api/v1/events/` — Listar eventos (filtrable por severity, resolved)
    - `POST /api/v1/events/` — Crear evento
    - `GET /api/v1/events/{id}/` — Detalle de evento
    - `PUT /api/v1/events/{id}/` — Actualizar evento

    **Health Checks:**
    - `GET /api/v1/health/` — Liveness (200 si el proceso vive)
    - `GET /api/v1/health/ready/` — Readiness (200 si DB+MQTT ok, 503 si no)

    Include for each endpoint: HTTP method, path, auth required (Yes/No), description, query params (for list endpoints), example response (brief).

    **Tópicos MQTT:**
    - `nodealert/{device_id}/telemetry` — Lecturas de sensores (publicado por ESP32 cada 10s)
    - `nodealert/{device_id}/events` — Eventos del sistema (publicado por ESP32 en transiciones)
    - `nodealert/{device_id}/commands` — Comandos del servidor (suscrito por ESP32)
    - `nodealert/{device_id}/status` — Heartbeat de estado (publicado por ESP32 cada 60s, QoS 1, con Last Will)

    Each topic: dirección (publish/subscribe), QoS, formato de payload.

    Minimum 100 lines.
