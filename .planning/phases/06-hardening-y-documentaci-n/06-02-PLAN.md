    ```python
    class LivenessHealthView(APIView):
        """GET /api/v1/health/ — Liveness check. No dependencies. Public."""
        permission_classes = [AllowAny]

        def get(self, request):
            return Response({"status": "alive"}, status=status.HTTP_200_OK)


    class ReadinessHealthView(APIView):
        """GET /api/v1/health/ready/ — Readiness check. Verifies MySQL + MQTT.

        Returns 200 if all dependencies healthy, 503 if any fail.
        Public endpoint — no sensitive data in response.
        """
        permission_classes = [AllowAny]

        def get(self, request):
            checks = {"database": False, "mqtt": False}

            # Check MySQL connectivity
            try:
                connection.ensure_connection()
                checks["database"] = True
            except Exception:
                pass

            # Check MQTT subscriber process is running (D-10)
            try:
                # pgrep returns 0 if mqtt_subscriber process found
                mqtt_alive = os.system(
                    'pgrep -f "mqtt_subscriber" > /dev/null 2>&1'
                )
                checks["mqtt"] = (mqtt_alive == 0)
            except Exception:
                pass

            if all(checks.values()):
                return Response(
                    {"status": "ready", "checks": checks},
                    status=status.HTTP_200_OK
                )
            return Response(
                {"status": "not ready", "checks": checks},
                status=status.HTTP_503_SERVICE_UNAVAILABLE
            )
    ```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet / client → nginx (port 80) | Untrusted HTTP requests enter the system |
| nginx → Django Gunicorn (port 8000, Docker internal) | Internal network, trusted after nginx validation |
| Docker socket → each container HEALTHCHECK | Agent check within Docker's control plane |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-04 | Denial of Service | Login endpoint | mitigate | AnonRateThrottle: 5 requests/min per IP on /api/v1/auth/login/ (D-14) |
| T-06-05 | Denial of Service | General API | mitigate | UserRateThrottle: 100 requests/min per authenticated user (D-14) |
| T-06-06 | Information Disclosure | Health endpoints | accept | Liveness/Readiness return only status + check booleans (alive/ready + database/mqtt booleans). No sensitive data, no stack traces, no version info |
| T-06-07 | Tampering | Django service | mitigate | Port 8000 exposure removed from docker-compose (D-07). Django only accessible via Docker internal network. nginx is the sole entry point |
| T-06-08 | Spoofing | nginx proxy headers | mitigate | X-Forwarded-For and X-Real-IP set by nginx. Django trusts these from internal proxy only |
| T-06-SC | Tampering | nginx:alpine image | mitigate | Official Docker image; verify checksum at first pull |
</threat_model>
