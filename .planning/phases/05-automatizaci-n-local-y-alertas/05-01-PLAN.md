**automation_manager.cpp** (`NodeAlert-Firmware/src/managers/automation_manager.cpp`):
Follow `task_manager.cpp` structural pattern:

- **Constructor**: Initialize all member pointers to `nullptr`, bools to `false`, `m_actuator_on_ms` to 0
- **`init()`**: Store queue handle and error handler pointer
- **`startTask()`**: `xTaskCreatePinnedToCore(automationTask, "auto_task", 4096, this, 2, &m_task_handle, 1)` — priority 2, stack 4096, core 1
- **`automationTask()`**: Static task function:
  - Cast `pvParams` to `AutomationManager*`
  - Local state: `SensorReading reading`, `bool alert_active = false`, `uint32_t alert_entry_time = 0`, `uint32_t actuator_on_time = 0`
  - `while(1)` loop:
    1. Non-blocking drain of sensor_queue: `while (xQueueReceive(self->m_sensor_queue, &reading, pdMS_TO_TICKS(0)) == pdTRUE)` → store latest 4 readings by type index (0-3) in `m_latest[4]`
    2. Get `now_ms = (uint32_t)(esp_timer_get_time() / 1000)`
    3. If NOT `m_override_active`:
       - Call `evaluateThresholds()` which compares each latest value against the corresponding threshold from `g_thresholds`
       - **Threshold evaluation logic per D-06/D-07**: For each sensor type, check if value exceeds warning/critical thresholds. Returns `true` if any critical threshold is crossed.
       - **Hysteresis logic** (D-07, dual AND condition — time AND delta):
         - On entry to alert (`any_critical && !alert_active`): set `alert_active = true`, `alert_entry_time = now_ms`, `actuator_on_time = now_ms`, call `controlActuator(true)`, call `publishEvent("threshold_crossed", "critical", "Critical threshold detected")`
         - On exit (`!any_critical && alert_active`): check BOTH `(now_ms - alert_entry_time) >= m_thresholds.hysteresis_time_ms` AND the delta condition — each sensor value must be below its warning threshold by the delta margin: `value < threshold * (1.0 - hysteresis_delta_pct)`. Only deactivate if BOTH conditions met. NEVER deactivate on delta alone.
         - When deactivating: `alert_active = false`, check `(now_ms - actuator_on_time) >= ACTUATOR_MIN_ON_MS`, if yes → `controlActuator(false)`, `publishEvent("actuator_off", "info", "Conditions normal")`
    4. Sleep `vTaskDelay(pdMS_TO_TICKS(3000))` per D-01

- **`evaluateThresholds()`**: Check each of the 4 sensor types against `g_thresholds`:
  - DHT22_TEMPERATURE > `g_thresholds.temp_critical` → critical
  - MQ9_GAS > `g_thresholds.gas_critical` → critical
  - KY026_FLAME > `g_thresholds.flame_threshold` → critical
  - DHT22_HUMIDITY < `g_thresholds.humidity_low` OR > `g_thresholds.humidity_high` → warning only
  - Also check warning thresholds for UI-level info (`temp_warning`, `gas_warning`), but only return critical boolean
  - Returns `true` if any critical condition is active

- **`controlActuator(bool turn_on)`**: 
  - If `turn_on === true`: `gpio_set_level(PIN_RELAY_ACTUATOR, 1)`, store `m_actuator_on_ms = now_ms`
  - If `turn_on === false`: In auto-mode (NOT override), enforce `(now_ms - m_actuator_on_ms) >= ACTUATOR_MIN_ON_MS` — if not elapsed, return early without turning off. In override mode, bypass the minimum time.

- **`publishEvent()`**: Use `esp_mqtt_client_publish()` to publish JSON to `nodealert/{DEVICE_ID}/events` topic with `device_id`, `event_type`, `severity`, `message`, `timestamp` keys. Include `extern esp_mqtt_client_handle_t` or access via a stored reference. Publish at QoS 1 per Phase 3 convention.

- **`processCommand(const char* json)`**: Use lightweight `strstr`-based parsing (per RESEARCH.md's recommendation — avoids cJSON heap):
  - `"cmd":"actuator_on"` → `m_override_active = true; m_override_actuator_on = true; controlActuator(true); publishEvent("override", "warning", "Override: Actuator ON")`
  - `"cmd":"actuator_off"` → `m_override_active = true; m_override_actuator_on = false; controlActuator(false); publishEvent("override", "warning", "Override: Actuator OFF")`
  - `"cmd":"return_to_auto"` → `m_override_active = false; publishEvent("override", "info", "Override: Return to Auto")`
  - `"cmd":"acknowledge_alarm"` → `m_alarm_acknowledged = true; publishEvent("override", "info", "Alarm acknowledged")`
  - `"cmd":"update_thresholds"` → parse params using `strstr`+`atof` for field names (`"temp_critical":`, `"gas_critical":`, etc.) and overwrite corresponding `g_thresholds` fields; `publishEvent("thresholds", "info", "Thresholds updated")`
